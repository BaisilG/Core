ReadCodepoint(Stream)

Reads a codepoint from the stream and advances the position within the stream by one codepoint. System.Array.Empty``1 signifies an invalid sequence or inability to read from the stream.

Returns

The codepoint as a System.Array of zero to two System.Char
Remarks

This method assumes UTF-8 encoding.

This method requires a bit of explaination as there's a great deal of misunderstandings. System.Char is a 16-bit integer, yet UNICODE requires at least 21 bits of storage. This is an archaic decision from back in the days of UCS-2. Changing things to update with the times wouldn't be feasible. Because of this, .NET strings sometimes require 2 characters to represent a single codepoint. That means, while in most cases, this operation will return a single character in an array, there are cases where the codepoint exceeds 16-bits and has to be encoded. For that reason, if you're on a new enough runtime, you want to use , which does this same operation but with a type capable of representing the codepoint as a single value.
